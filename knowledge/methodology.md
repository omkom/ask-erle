# How ask-erle Was Built

## The Approach: AI-Orchestrated, Human-Directed

ask-erle was co-created with Claude Code. This is deliberate and worth explaining.

At Erlé's level of practice, the value is not in typing code line by line — it's in **architecture decisions, quality control, and orchestration**. The same way a VP Engineering doesn't write every line of production code but designs the system and ensures quality, Erlé orchestrates AI agents to produce work that meets his standards.

### What Erlé Did (Human)
- Designed the agent architecture (single-skill RAG, no overengineering)
- Wrote the behavioral constitution (CLAUDE.md rules that enforce honesty and grounding)
- Curated and verified every fact in the knowledge files
- Made strategic decisions: what to include, what to emphasize, what gaps to acknowledge
- Directed the iterative refinement through multiple working sessions
- Quality-controlled every output against consulting-grade standards

### What Claude Code Did (AI)
- Structured the markdown knowledge files from raw career data
- Generated the initial SKILL.md routing logic
- Produced the README documentation
- Cross-checked consistency across files

### Why This Matters
This is exactly the workflow Erlé proposes for engineering teams: humans set direction, define quality bars, and make judgment calls. AI handles execution, formatting, and consistency checks. The result is 5x output at the same quality — proven by 14 enterprise audit deliverables produced solo that matched a 5-person consulting team's output.
